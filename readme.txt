Заметки по работе приложения:

	Для входа в приложение Scope_lap и работы с заполнением базы данных nurngburg можно использовать пользователей postrgres и stig. 
	Логины пользователей я привязал к сущностям таблицы drivers, указав их (логины) в столбце driver_login таблицы drivers, таким образом попытавшись 
	сформировать аккаунты пользователей.

	В базе данных я создал различные представления, процедуры и функции, чтобы в самом приложении вызывать именно их, а не делать прямые запросы 
	непоспредственно к таблицам базы данны. Для вызова этих запросов я создал отдельный класс Commands, в котором через статические методы вызывал команды,
	необходимы для работы.

	В приложении при выводятся всё гоночные сессии всех пользователей в DataGrid от самых новых до самых старых записей (по дате гонончной сессии).
	Кнопка "Your sessions" выводит сессии текущего пользователя, "View top 100" - самые быстрые сессии, "All drivers track session" - все сессии.

	Пользователь может добавлять новые сессии, которые он провел. Создавать сессии для других водителей он не может, как и редактировать либо удалать чужие сессии.
	Пользователь может редактировать свои сессии, однако на данном этапе я не доработал функционал - в сессии нельзя поменять только один пункт - нужно перезаполнять
	все требуемые поля.
